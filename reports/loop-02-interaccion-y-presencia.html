Decisiones del equipo (5 propuestas → síntesis)

  • enviada Tooltip táctil (optimizador de dashboards): toca/arrastra el chart y ves el valor de cada serie en ese instante.
  • enviada Punto "ahora" pulsante (diseñador): halo vivo en la presencia actual de cada serie.
  • enviada "Activo ahora" en Inicio (UX): chips con % de presencia + deep-link a Cuerpo.
  • enviada Línea de referencia 25% (analista, reencuadrada): guía de lectura neutral, sin lenguaje de re-dosis.
  • diferida Modelo de absorción bifásico GLP-1 (investigador): científicamente valioso pero cambia el modelo core y sólo hay datos para pocos productos → loop futuro.
  • descartada "Re-dosis sugerida" (analista): sugerir cuándo re-dosificar = consejo de dosificación → incumplimiento. No se implementa.

Qué se construyó

  • Tooltip: una capa <rect> sobre el plot captura el pointer, mapea clientX → tiempo, interpola cada serie y dibuja guía vertical + puntos + caja con valores. touchAction: pan-y deja scrollear vertical.
  • Pulso "ahora": motion.circle con halo r:4→9, opacity:0.35→0 en loop; respeta prefers-reduced-motion.
  • Activo ahora (Inicio): presenceNow(state, now) reusa el modelo de decaimiento; chips de productos con presencia ≥1%, tap → setProgresoView('cuerpo') + tab Progreso.
  • Referencia 25%: línea horizontal punteada neutra (solo en modo %).
  • Vista de Progreso global: el segmento (Calendario/Avances/Cuerpo) pasó a state.progresoView para permitir el deep-link desde Inicio.

Red-team Codex + self-audit

  • Codex Sin bugs concretos. tsc --noEmit y build limpios.
  • self-audit Mapeo del tooltip exacto: width:100% + viewBox sin alto fijo conserva el ratio (sin letterbox).
  • self-audit Pulso ligero (1 círculo/serie) con guard de motion; tooltip cabe; deep-link aplica ambas acciones; presenceNow sin división por cero, pct ≤ 100.
  • cumplimiento 25% es referencia neutral; "Activo ahora ~X%" es presencia de datos propios; disclaimers en Cuerpo. Sin consejo de dosis.
